# Understanding the Build Process

## Template Repository

The template repository for this assignment is: [understanding-build-process](https://github.com/Ivy-Tech-SDEV243-Assignment-Templates/understanding-build-process).

## Introduction

The purpose of this assignment is to become familiar with the technologies that are used in the course, and the assignment submission process. All future assignments will use foundational skills established in this assignment.

This assignment has a full video walkthrough to accompany the written instructions. You simply need to follow along and complete the steps as described. Be sure to match the video output exactly, including:

- filenames
- code
- comments
- commit messages
- screenshots

Future assignments may require you to complete additional checkpoints on your own, so make sure to double check the requirements in the README.md file for each assignment.

## Learning Goals

- Copy a repository from a template repository.
- Create a codespace for a repository (or open the repository locally)
- Understand the file and directory structure for projects in this course.
- Compile a .NET application using the `dotnet` CLI.
- Examine the role of build artifacts in a build process.
- Understand which parts of an application should be included in source control.
- Run a .NET application using the `dotnet` CLI.
- Commit and push checkpoint milestones to the repository.
- Confirm that changes show up in the remote github repository
- Submit the URL of an assignment repository
- Deallocate a codespace (if you are using them)
